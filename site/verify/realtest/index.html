    下板验证说明

    1. RTL修改

    1.1 DRAM访存地址修改

      用于下板测试的汇编程序采用IROM和DRAM统一编址，因此需要对原DRAM的访存地址进行修改：

      原DRAM代码：

    dram U_dram (
    .clk    (cpu_clk),
    .a      (waddr[15:2]),
    .spo    (rdata),
    .we     (we),
    .d      (wdata)
);

      修改后的DRAM代码：

    wire [31:0] waddr_tmp = waddr - 16'h4000;

dram U_dram (
    .clk    (cpu_clk),
    .a      (waddr_tmp[15:2]),
    .spo    (rdata),
    .we     (we),
    .d      (wdata)
);

    1.2 数码管显示修改

      原本数码管显示PC，现修改为显示x19寄存器的值。

    2. 存储器IP替换

      由于下板测试程序较大，导致导入.coe后重新综合工程耗时较长，因此课程指导书网站提供了已综合好的IROM和DRAM IP核，同学们可将其直接导入CPU工程中使用。

      下面以导入DRAM为例，介绍导入已综合IP核的方法 (IROM同理)。

      首先，从指导书网站下载download_test.zip压缩包文件并解压到无中文路径下。

      然后，备份原工程，按照上一节所述修改RTL代码，并删除已实例化的IROM和DRAM IP核。

    4. 下板结果说明

      每通过一个测试点，数码管显示的数值会加1，直至测试完毕。测试完毕时，数码管将以16进制形式显示通过测试的指令数 (例如，如果实现了单周期的35条指令并通过了测试，则数码管将显示0x23)。

      需要注意的是，对于测试不通过的情形，如果数码管显示的值停在n，则表示第n+1条指令的测试失败。测试失败时，可查看start.dump的测试代码以定位出错指令，并进行相应的调试。
