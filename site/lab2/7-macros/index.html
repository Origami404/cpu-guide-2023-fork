    宏定义使用

      不管是使用高级语言进行软件开发，还是使用HDL进行硬件设计，都要尽量遵循代码可配置的原则。对于常量或参数，应当在代码中使用宏定义的方式来引用。使用宏定义，不仅可以保证代码具有良好的可配置性，还能提高代码的规范性和可读性。

      我们在“控制单元设计”的一节中，设计了如表5-2所示的控制信号取值表。应该注意到，明明是控制信号的取值表，表中的npc_op和alu_op却用字符串作为信号取值，显得“格格不入”。事实上，npc_op和alu_op的取值就是以宏定义的方式来表示的。将npc_op、alu_op与其他控制信号相比，可以发现宏定义的表达方式不仅更方便阅读、更容易为人所理解，而且有利于代码维护。

      下面简要介绍如何在Verilog HDL中使用宏定义：

      (1) 新建一个独立的.v文件 (如param.v)，用来编写宏定义；

      (2) 使用`define关键字定义宏参数，如图8-1所示。

    // file: param.v
`ifndef CPU_PARAM
`define CPU_PARAM

    // syntax: `define <macro name> <parameter>
    `define ADD     'b000
    `define SUB     'b001
    `define OR      'b010
    `define AND     'b011
    ......

    `define STATE_IDLE 'b0001
    `define STATE_WRIT 'b0010
    `define STATE_WORK 'b0100
    `define STATE_RETU 'b1000
    ......

`endif
    图8-1 Verilog HDL定义宏参数的语法

      (3) 有2种方法可以引用param.v：

        1) 在Vivado中，将宏定义文件设置成全局引用，如图8-2所示。

    图8-2 设置宏定义文件为全局引用

        2) 在需要调用参数的源文件中使用`include "param.v"引用，如图8-3所示。 

    `include "param.v"

    ......

    state_init <= `STATE_IDLE;
